## Red-team pass — what it tried and found
A separate subagent reviewed the code without having written it. Findings and fixes:

| ID | Severity | Finding | Fix |
|----|----------|---------|-----|
| C1 | Critical | postMessage origin check broken under sandbox (origin is `"null"`, all messages dropped) | Switched to `event.source` verification against iframe's `contentWindow` |
| H1 | High | Client wrote a non-HttpOnly duplicate device-id cookie | Removed client-side cookie write; server middleware is the sole cookie setter |
| H2 | High | Client and server could end up with different device-ids | Same fix as H1 — client only mirrors in localStorage, doesn't set cookies |
| M1 | Medium | Manifest validation too loose (no type/enum/non-empty checks) | Added per-field validation: non-empty strings, controls enum, tags string-array |
| M2 | Medium | `entry` field unvalidated → path traversal | Added regex `^[A-Za-z0-9._-]+\.html?$` on entry |
| M3 | Medium | SAVE_WRITE slot/data unbounded | Slot name regex + 1 MB data cap, returns `false` on violation |
| M4 | Medium | Early game messages dropped before IDB store ready | Uses no-op store fallback; READY/HEARTBEAT always handled |
| M5 | Medium | `onSaveWrite` rejections unhandled | `try/catch` with console.warn |

Areas verified OK by red-team: INV-S1 sandbox tokens, INV-S3 ad slot isolation, INV-S5 namespace escape (IndexedDB keys are opaque strings), INV-S7 no unsafe-eval, INV-F1 house-ad timing, prototype pollution (structured clone, no merge).

## Unfixed edge cases / follow-ups
- **L1 — `unsafe-inline` in script-src**: weakens CSP against XSS. Moving to nonce-based CSP is a future effort (Next.js supports nonces via middleware). Not blocking for v1.
- **L2 — No `Secure` flag on device cookie**: should be added when site is on HTTPS in production (Vercel enforces HTTPS, but the cookie flag should match). Not blocking for local dev.
- **L3 — AdSlot `onError` is dead code**: iframe `onError` doesn't fire for no-fill. The 500ms timer is the real fallback. Cleanup deferred.
- **L4 — Heartbeat-only games can avoid "not responding"**: a game with a dead render loop but a live `setInterval(HEARTBEAT)` won't trigger the affordance. Matches the literal invariant wording; deferred.
- **S2–S7 issues were incidentally built by Cursor but NOT verified against their own acceptance criteria.** They need their own `/part2` passes. Only S1's gate is green.
- **Device-id sync (H2 root cause)**: the server-set HttpOnly cookie and the client-side localStorage id can still diverge. A proper fix needs the server to render the device-id into a client-readable (but game-inaccessible) channel. Deferred to a follow-up issue.
